# Требования к проекту VinWin FAQ

> Статус: **валидировано v1.0**  
> Дата: 2026-05-09

---

## Контекст

Статический сайт на GitHub Pages с ответами на базовые вопросы про нейронные сети и LLM. Цель — перестать отвечать на одни и те же вопросы вручную, давать ссылку вместо объяснения.

---

## Функциональные требования (FR)

| ID | Требование | Приоритет |
|----|-----------|-----------|
| FR-01 | Сайт отображает Q&A в двух режимах: **одна длинная страница** и **многостраничный** (по категориям) | Must |
| FR-02 | Явная кнопка/переключатель смены режима отображения, видна всегда; при смене режима позиция (открытый вопрос/категория) сохраняется | Must |
| FR-03 | В режиме одной страницы вопросы раскрываются аккордеоном | Must |
| FR-04 | В многостраничном режиме — навигация по категориям, каждая категория отдельный раздел | Must |
| FR-05 | Категории (минимум на старте): «Что такое» (базовые понятия) и «Ограничения и мифы» | Must |
| FR-06 | Весь контент на русском языке | Must |
| FR-07 | Начальный объём: 5–15 вопросов; структура должна легко расширяться | Must |
| FR-08 | Навигационное меню с якорями/ссылками на категории | Should |
| FR-09 | Каждый вопрос имеет уникальный якорь `#id` на главной странице (для шаринга конкретного ответа) | Should |

---

## Нефункциональные требования (NFR)

| ID | Требование | Критерий |
|----|-----------|----------|
| NFR-01 | Хостинг на GitHub Pages (статика) | Нет серверного кода |
| NFR-02 | Дизайн — лёгкий, дружелюбный, рассчитан на новичков | Нет жаргона в UI, читаемые шрифты |
| NFR-03 | Desktop-first; на мобиле не ломается, но не оптимизируется специально | Viewport meta, базовый responsive |
| NFR-04 | Zero-pipeline: правишь HTML-файл — сайт сразу работает без шага сборки. Локальная разработка в git, публикация через push на GitHub Pages | Нет build-инструментов, нет CI |
| NFR-05 | Быстрая загрузка: < 1 сек на типичном соединении | Без тяжёлых фреймворков, без CDN-зависимостей |
| NFR-06 | Одиночный автор; CMS и система авторизации не нужны | — |

---

## Ограничения

- Только статическая генерация (GitHub Pages не выполняет серверный код)
- Без внешних зависимостей на runtime (или минимум, без node_modules в production)
- Один мейнтейнер; разработка локально в git, деплой через push в GitHub Pages

---

## Вне скоупа (явно исключено)

- Поиск по сайту
- Мультиязычность
- Комментарии / обратная связь от пользователей
- Аналитика
- Тёмная тема

## Стек (зафиксировано)

**Чистый HTML + CSS + Vanilla JS** — единственный файл `index.html` (или небольшой набор файлов).

- Нет зависимостей, нет build-шага
- Правишь файл локально → `git push` → GitHub Pages публикует немедленно
- Добавление вопроса = вставить блок HTML в нужное место
